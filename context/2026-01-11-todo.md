# This is desired cli wizard design

`*` means text that is printed on screen above menu options
`**` means just text on the screen, like a `man` page
`-` menu options that user can pick (exlcuding common like `back` etc)
`+` special notes

- Main menu
  * Failed checks X/YY
  * Required - 6 (hidden if empty)
  * Recommended - 12 (hidden if empty)
  * Opinionated - 21 (hidden if empty)
  - Current issues
    * Same as in main screen
    - Overview
      ** shows detailed overview about all failed checks
    - Fix issues
      + this screens repeats for each issue in a queue
      * Info about first issues in a queue
      - Accept auto fix
      - Why?
      - Next
      - Mute for 2 weeks
      - Mute for 2 months
      - Disable
      ** in the end of queue show summary of current run, example: 3 issues fixed, 3 temporarily muted, 4 disdabled, 7 remained
  - Config
    - About config
      ** info about main categories (required, recommended etc.), tags (nodejs, common etc) and checks bound to tool etc
    - Control categories
      - checkbox for category 1
      - checkbox for category 2
      ...
      - checkbox for category x
